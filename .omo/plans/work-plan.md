## 关键决策点

1. **Redis 前置**：将 Redis 集成提前到 Phase 1，因为后续多个功能依赖 Redis（限流、缓存、任务调度）
2. **多租户简化**：不新建 Tenant 模型，直接在现有 Organization 基础上增加租户上下文中间件
3. **功能合并**：合并重复功能（消息推送、邮件发送等），减少冗余开发
4. **优先级调整**：先完成基础设施和安全相关功能，再开发业务功能

## 风险评估

| 风险 | 影响 | 缓解措施 |
|------|------|---------|
| Redis 集成复杂度 | 高 | 使用成熟库（ioredis），分步实施 |
| 多租户数据隔离 | 中 | 先在 UserList 中试点，再推广到其他模块 |
| WebSocket 连接稳定性 | 中 | 实现自动重连机制，监控连接状态 |
| 第三方登录 OAuth 配置 | 低 | 先实现 GitHub，再扩展其他平台 |
| 移动端 App 兼容性 | 中 | 使用 Expo 跨平台框架，减少原生代码 |

## 验收标准

1. 所有 typecheck 通过，无类型错误
2. 所有 lint 检查通过，无警告
3. 核心功能手动 QA 通过
4. 无 Must NOT 功能被实现
5. 所有功能独立可交付，可单独部署
